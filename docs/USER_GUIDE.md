## 1. 平台概述

### 1.1 什么是 AWD Arena

AWD Arena 是一个现代化的 AWD (Attack with Defense) 攻防对抗竞赛管理平台，专为网络安全竞赛设计。

### 1.2 核心功能

| 功能模块 | 说明 |
|---------|------|
| 🏆 比赛管理 | 创建、配置、启动、监控比赛 |
| 👥 队伍系统 | 队伍注册、成员管理、口令加入 |
| 📊 计分系统 | 实时攻击分/防御分、零和计分、首杀奖励 |
| 🐳 容器管理 | 自动生成靶机、状态监控、一键重启 |
| 📈 排行榜 | 实时排名、数据导出、趋势分析 |
| 🔒 安全审计 | 操作日志、权限控制、XSS防护 |

### 1.3 技术架构

- **后端**: Go + Fiber + GORM + WebSocket
- **前端**: React + TypeScript + Ant Design
- **数据库**: SQLite / PostgreSQL / MySQL
- **容器**: Docker SDK

## 4. 管理员指南

### 4.1 用户管理

#### 创建用户

1. 进入 **管理后台 → 用户管理**
2. 点击 **创建用户**
3. 填写用户信息：
   - 用户名（必填）
   - 密码（必填）
   - 角色（选择：admin/organizer/player/observer）
   - 所属队伍（可选）
4. 点击 **保存**

#### 重置密码

1. 在用户列表找到目标用户
2. 点击 **重置密码**
3. 输入新密码
4. 通知用户新密码

### 4.2 比赛管理

#### 创建比赛

1. 进入 **管理后台 → 比赛管理**
2. 点击 **创建比赛**
3. 填写比赛配置：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 比赛名称 | 比赛标题 | - |
| 比赛描述 | 比赛说明 | - |
| 比赛模式 | awd_score / classic | awd_score |
| 轮次时长 | 每轮时间（秒） | 300 |
| 休息时长 | 轮次间隔（秒） | 120 |
| 总轮次 | 比赛轮数 | 20 |
| Flag格式 | Flag模板 | flag{%s} |
| 攻击权重 | 攻击分系数 | 1.0 |
| 防御权重 | 防御分系数 | 0.5 |

4. 点击 **创建**

#### 比赛生命周期

```
  ┌─────────┐
  │  准备   │ ← 比赛创建后的初始状态
  └────┬────┘
       │ 启动
       ↓
  ┌─────────┐    暂停    ┌─────────┐
  │  进行中  │ ←───────→ │   暂停   │
  └────┬────┘           └─────────┘
       │ 结束
       ↓
  ┌─────────┐
  │   结束   │ ← 比赛完成，不可恢复
  └─────────┘
```

#### 操作说明

| 操作 | 说明 | 快捷方式 |
|------|------|----------|
| 启动比赛 | 开始比赛，生成容器 | POST /api/v1/games/:id/start |
| 暂停比赛 | 暂停计分和容器 | POST /api/v1/games/:id/pause |
| 恢复比赛 | 从暂停恢复 | POST /api/v1/games/:id/resume |
| 结束比赛 | 结束比赛，计算最终排名 | POST /api/v1/games/:id/end |
| 删除比赛 | 删除比赛及所有数据 | DELETE /api/v1/games/:id |

### 4.3 Docker 镜像管理

#### 查看宿主机镜像

1. 进入 **Docker 镜像管理**
2. 点击 **宿主机镜像** 标签
3. 查看所有可用镜像

#### 拉取镜像

1. 点击 **拉取镜像**
2. 输入镜像名称（如：`nginx:latest`）
3. 等待拉取完成

#### 删除镜像

1. 在镜像列表选择要删除的镜像
2. 点击 **删除**
3. 确认删除

⚠️ **注意**: 正在使用的镜像无法删除

### 4.4 审计日志

#### 查看日志

1. 进入 **审计日志** 页面
2. 可按以下条件筛选：
   - 时间范围
   - 操作类型
   - 用户
3. 支持导出 CSV/PDF

#### 日志类型

| 类型 | 说明 |
|------|------|
| login | 用户登录 |
| logout | 用户登出 |
| create | 创建操作 |
| update | 更新操作 |
| delete | 删除操作 |
| flag_submit | Flag提交 |

- 容器健康状态

#### 容器管理

1. 进入比赛详情 → **容器管理**
2. 查看所有队伍容器状态
3. 可执行操作：
   - 重启单个容器
   - 批量重启
   - 查看容器日志

### 5.3 比赛结算

#### 查看排行榜

1. 进入 **排行榜** 页面
2. 查看实时排名
3. 可导出排名数据

1. 进入比赛详情

3. 选择格式：PDF / CSV / Excel

## 6. 选手指南

### 6.1 账号与队伍

#### 注册账号

1. 访问注册页面
2. 填写注册信息：
   - 用户名
   - 密码
   - 队伍口令（可选）
3. 点击 **注册**

#### 加入队伍

**方式一**: 注册时输入队伍口令自动加入

**方式二**: 登录后操作
1. 进入 **个人中心**
2. 点击 **加入队伍**
3. 输入队伍口令
4. 确认加入

### 6.2 参赛流程

#### 步骤1: 查看比赛信息

1. 登录后进入 **比赛列表**
2. 查看正在进行或即将开始的比赛
3. 点击比赛查看详情

#### 步骤2: 获取靶机信息

1. 进入比赛详情页
2. 查看 **我的靶机** 信息
3. 记录以下信息：
   - IP地址
   - 端口
   - SSH账号密码

#### 步骤3: 攻击其他队伍

1. 扫描其他队伍靶机
2. 发现漏洞
3. 获取Flag
4. 提交Flag

#### 步骤4: 提交Flag

1. 点击 **提交Flag** 按钮
2. 输入获取的Flag
3. 点击 **提交**
4. 查看提交结果

### 6.3 比赛技巧

#### 攻击策略

- 🔍 **信息收集**: 先扫描目标，了解服务类型
- 🎯 **批量攻击**: 使用脚本批量攻击多个队伍
- ⏰ **时机选择**: 每轮开始时第一时间攻击
- 📝 **记录经验**: 记录成功的攻击方法

#### 防御策略

- 🔒 **及时修补**: 发现漏洞立即修复
- 📊 **监控日志**: 监控异常访问
- 🔄 **定期备份**: 备份关键文件
- 🛡️ **加固服务**: 关闭不必要的服务

#### Flag 提交最佳实践

- ✅ 复制粘贴，避免手动输入错误
- ✅ 提交前确认Flag格式正确
- ✅ 同一Flag不要重复提交
- ✅ 批量获取后尽快提交

## 7. 功能模块详解

### 7.1 排行榜系统

#### 积分规则

**零和计分**:
- 攻击成功：攻击者得分，被攻击者失分
- 分数 = 基础分 × 攻击权重

**首杀奖励**:
- 每个挑战的第一个成功攻击额外加分
- 首杀奖励通常为基础分的2倍

#### 排名规则

1. 总分 = 攻击分 + 防御分 + 首杀奖励
2. 总分相同时，按攻击分排序
3. 攻击分也相同，按首杀数排序

### 7.2 Flag 系统

#### Flag 格式

标准格式：`flag{xxxxxxxxxxxx}`

示例：
- `flag{sql_injection_master}`
- `flag{xss_reflected_2024}`

#### Flag 生成机制

- 每轮开始时为每个队伍生成唯一Flag
- Flag 基于队伍ID、轮次、密钥生成
- 每个Flag只在当前轮次有效

### 7.3 容器系统

#### 容器分配

```
比赛启动
    │
    ├── 为队伍A生成容器
    │   ├── 靶机1 (Port: 8001)
    │   └── 靶机2 (Port: 8002)
    │
    ├── 为队伍B生成容器
    │   ├── 靶机1 (Port: 8003)
    │   └── 靶机2 (Port: 8004)
    │
    └── ...
```

#### 端口映射

- 宿主机端口范围：8000-9000
- 自动分配，避免冲突
- 映射信息可在容器管理页面查看

### 7.4 WebSocket 实时通信

#### 连接状态

- 🟢 已连接：实时接收更新
- 🔴 已断开：页面刷新或网络问题

#### 消息类型

| 类型 | 说明 |
|------|------|
| ranking_update | 排行榜更新 |
| flag_submit | Flag提交结果 |
| round_change | 轮次变化 |
| container_status | 容器状态变化 |

## 8. 常见问题 FAQ

### Q1: 忘记密码怎么办？

**A**: 联系管理员重置密码

### Q2: Flag 提交后显示"已过期"？

**A**: Flag 只在当前轮次有效，下一轮会生成新Flag

### Q3: 无法连接到靶机？

**A**: 
1. 检查容器是否运行（查看容器状态）
2. 检查端口是否正确
3. 联系裁判处理

### Q4: 排行榜没有实时更新？

**A**: 
1. 检查WebSocket连接状态
2. 刷新页面
3. 检查网络连接

### Q5: 移动端页面显示不正常？

**A**: 
1. 使用Chrome/Safari最新版本
2. 点击左上角汉堡菜单展开侧边栏
3. 建议横屏使用

### Q6: 如何退出队伍？

**A**: 
1. 进入个人中心
2. 点击 **退出队伍**
3. 确认退出

### Q7: 容器被重启了，Flag还在吗？

**A**: Flag会自动重新注入，但建议攻击前确认

### Q8: 比赛结束了还能提交Flag吗？

**A**: 不能，比赛结束后Flag提交功能关闭

## 9. 最佳实践

### 9.1 管理员最佳实践

#### 安全建议

- ✅ 定期修改管理员密码
- ✅ 限制管理后台访问IP
- ✅ 定期检查审计日志
- ✅ 备份数据库
- ✅ 使用HTTPS

#### 性能优化

- 使用PostgreSQL替代SQLite（大型比赛）
- 定期清理历史比赛数据
- 监控服务器资源使用

### 9.2 裁判最佳实践

#### 比赛组织

- 提前30分钟开始比赛准备
- 准备应急预案（容器故障、网络问题）
- 实时监控比赛状态
- 记录异常情况

#### 公平性保障

- 确保所有队伍容器配置一致
- 公布Flag格式和计分规则
- 及时处理申诉

### 9.3 选手最佳实践

#### 比赛前

- 熟悉平台操作
- 准备常用工具
- 测试网络连接

#### 比赛中

- 保持冷静，合理分配时间
- 攻防并重
- 与队友协作

#### 比赛后

- 复盘总结
- 整理WriteUp
- 分享经验
