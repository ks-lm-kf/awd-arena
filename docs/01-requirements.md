# AWD 智能靶场平台 — 需求分析文档

> 版本: v1.0 | 日期: 2026-03-19 | 作者: PM Agent

---

## 1. 产品定位

### 1.1 目标用户

| 用户群体 | 典型场景 | 核心诉求 |
|---------|---------|---------|
| **CTF 选手** |赛前训练、战队内训、赛后复盘| 高仿真环境、自动化评分、实时排名 |
| **培训机构** | 网安教学、实训考核、认证培训 | 教学管理、学员追踪、一键部署回收 |
| **企业内训** |红蓝对抗、安全岗培训、应急演练| 规模化并发、权限管理、审计日志 |

### 1.2 核心价值

- **开箱即用**：一键部署完整 AWD 竞赛环境，无需手动配置 Docker/网络/靶机
- **智能攻防**：AI 辅助漏洞检测、流量分析、加固建议，降低使用门槛
- **真实仿真**：模拟真实 CTF 比赛流程，支持经典 AWD / AWDP / 自定义模式
- **教学友好**：集成学员管理、成绩追踪、操作回放，适合教学培训
- **跨平台**：Windows / Linux 双平台支持，降低部署门槛

### 1.3 产品名称

**AWD Arena** — 智能 AWD 靶场平台

## 2. 核心功能模块

### 2.1 竞赛管理

#### 2.1.1 赛事管理
- 创建/编辑/归档赛事
- 赛事模板（CTF标准赛、教学训练赛、企业内训赛）
- 赛事状态机：待开始 → 进行中 → 暂停 → 已结束

#### 2.1.2 队伍管理
- 队伍注册（手动/批量导入/邀请码）
- 队员管理（每队 3-5 人）
- 队伍信息展示（队名、Logo、所属单位）
- 支持个人赛模式（单人参赛）

#### 2.1.3 积分排名
- 实时积分排行榜（Web 刷新 + WebSocket 推送）
- 多维度排名：总积分 / 攻击得分 / 防御得分 / 综合排名
- 积分规则可配置（攻击得分权重、防御得分权重、Check 通过率）
- 排名历史曲线（按 Round 记录排名变化）

#### 2.1.4 Round 机制
- 可配置 Round 时长（默认 2 分钟）
- Round 自动推进，支持暂停/手动推进
- 每个 Round 结束时自动 Check + 计分
- Round 状态实时展示

### 2.2 靶场环境

#### 2.2.1 Docker 容器管理
- 靶机镜像管理（上传、导入、版本管理）
- 容器生命周期：创建 → 启动 → 运行 → 重启 → 销毁
- 容器资源限制（CPU、内存、网络带宽）
- 容器健康检查（自动检测存活状态）
- 容器日志查看（实时 + 历史日志）

#### 2.2.2 靶机分配
- 队伍与靶机的一对一绑定
- 每队分配一台攻击靶机（被攻击目标）+ 可选攻击机
- 靶机网络隔离（VLAN / Docker Network）
- 靶机端口映射（SSH、Web、数据库等）
- 赛前环境一致性校验（MD5 / 文件 hash）

#### 2.2.3 Flag 机制
- 动态 Flag：每轮自动更换，写入指定位置
- Flag 格式可配置（默认 `flag{uuid}`）
- Flag 哈希校验（防篡改检测）
- Check 脚本验证服务可用性（HTTP 200 / 特定返回内容 / 数据库连接）
- 被攻破后自动标记、扣分、生成攻击事件

### 2.3 攻击面板

#### 2.3.1 Flag 提交
- 攻击面板入口，支持提交 Flag
- 批量提交（一次提交多个 Flag）
- Flag 提交结果反馈（正确/错误/重复/已过期）
- 提交历史记录

#### 2.3.2 自动化攻击脚本
- 内置常用攻击脚本库（SQL注入、XSS、命令注入、文件包含、反序列化等）
- 脚本市场（用户上传/分享攻击脚本）
- 脚本一键执行（选择目标 IP + 脚本）
- 脚本执行结果展示（成功/失败/耗时）

#### 2.3.3 流量监控
- 实时流量抓取（tcpdump / mitmproxy）
- 流量协议解析（HTTP/HTTPS/DNS/TCP）
- 攻击流量标记与分类（SQLi、XSS、RCE、目录遍历等）
- 流量回放功能
- PCAP 文件导出

### 2.4 防御面板

#### 2.4.1 流量检测
- 入站流量实时分析
- 异常流量告警（高频请求、异常 User-Agent、SQL 注入特征、Webshell 特征）
- 流量日志与搜索

#### 2.4.2 WAF（Web Application Firewall）
- 内置轻量级 WAF 规则引擎
- 预设规则集（OWASP Top 10）
- 自定义规则（正则匹配、IP 黑名单、请求频率限制）
- WAF 日志与拦截统计

#### 2.4.3 加固建议
- AI 分析靶机配置，自动生成加固建议
- 常见加固项：弱密码、默认配置、危险函数、未授权访问
- 一键加固脚本（可选，风险提示）

#### 2.4.4 自动 Patch
- 赛前预置 Patch 脚本
- 比赛中一键应用补丁
- Patch 历史记录与回滚
- Patch 验证（应用后 Check 是否通过）

### 2.5 智能化功能

#### 2.5.1 AI 辅助攻防
- **攻击建议**：AI 分析靶机服务指纹，推荐可能的攻击路径
- **防御建议**：AI 分析攻击流量，推荐防御策略和加固方案
- **赛况分析**：AI 生成比赛局势分析报告（谁在领先、攻击趋势、防御薄弱点）

#### 2.5.2 自动流量分析
- 基于 ML 模型的流量异常检测
- 自动分类攻击类型
- 零日攻击模式发现

#### 2.5.3 异常检测
- 靶机服务异常检测（端口挂掉、服务崩溃、CPU/内存异常）
- 异常自动告警（平台通知 + 靶机通知）
- 异常自动恢复（可选：自动重启容器）

### 2.6 自动化功能

#### 2.6.1 一键部署
- `awd deploy --config config.yaml` 一键部署完整环境
- 自动拉取镜像、创建容器、配置网络、启动服务
- 环境自检（端口冲突、资源不足、Docker 可用性）

#### 2.6.2 自动评分
- Round 结束自动 Check 所有靶机
- 自动计算积分（攻击得分 + 防御得分 + Check 通过奖励）
- 积分变更实时推送

#### 2.6.3 自动恢复
- Check 失败自动重启容器
- 被攻破后自动恢复 Flag（重置为最新 Flag）
- 恢复次数限制与扣分机制

#### 2.6.4 轮次管理
- 自动推进 Round
- Round 间 Flag 轮换
- 赛中暂停/恢复
- 比赛倒计时

### 2.7 监控大屏

#### 2.7.1 实时排名
- 排行榜实时刷新（WebSocket 推送）
- 排名变化动画（上升/下降箭头）
- 积分分布图

#### 2.7.2 攻击可视化
- 攻击关系图（谁攻击了谁，攻击类型）
- 攻击热力图（按时间/队伍/靶机）
- 攻击事件流（实时滚动展示最新攻击事件）

#### 2.7.3 流量图表
- 全网流量曲线
- 各队伍流量对比
- 协议分布饼图
- 异常流量标记

## 3. 竞赛模式

### 3.1 经典 AWD 模式
- 每队一台靶机，同时攻击他人、防御自己
- 动态 Flag，每轮轮换
- 积分 = 攻击得分 + 防御得分（Check 通过奖励）+ 罚分
- Check 机制验证服务可用性

### 3.2 AWDP 模式（Attack With Defense Patch）
- 在经典 AWD 基础上增加**漏洞修补**环节
- 每轮发布新的漏洞（Patch 题目）
- 选手需要：① 漏洞利用攻击 ② 编写 Patch 修补自己的靶机
- Patch 审核机制：平台自动验证 Patch 是否修复了漏洞且不影响正常功能
- 积分 = 攻击得分 + Patch 得分 + 防御得分

### 3.3 自定义模式
- 自定义积分规则（攻击/防御/Check 权重）
- 自定义 Round 时长
- 自定义 Check 脚本
- 自定义 Flag 格式
- 自定义靶机镜像
- 支持混合模式（AWD + AWDP）

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 并发队伍数 | 最低 50 队，目标 200 队 |
| Flag 提交响应 | < 500ms |
| 排行榜刷新 | < 1s（WebSocket 推送） |
| Check 并发 | 所有靶机 30s 内完成 |
| 监控大屏 FPS | ≥ 30fps |
| 平台可用性 | 99.9%（比赛期间） |

### 4.2 安全要求
- 平台与靶机网络隔离
- Flag 加密存储与传输（AES-256）
- 用户密码 bcrypt 哈希存储
- API 接口 JWT 认证 + RBAC 权限控制
- 操作审计日志（不可篡改）
- XSS/CSRF/SQL 注入防护
- 敏感配置环境变量化

### 4.3 跨平台要求
- **服务器端**：Linux（推荐 Ubuntu 22.04+），Windows（WSL2 兼容）
- **管理端**：Web 界面，浏览器无关（Chrome/Firefox/Edge）
- **CLI 工具**：Python 跨平台 CLI（`awd-cli`）
- Docker 为核心运行时，Windows 需安装 Docker Desktop

### 4.4 部署要求
- **一键部署**：单脚本完成安装（`install.sh` / `install.ps1`）
- **离线部署**：支持离线镜像包导入（网络受限环境）
- **环境要求**：
  - CPU: ≥ 4 核
  - 内存: ≥ 8GB（推荐 16GB）
  - 磁盘: ≥ 50GB SSD
  - Docker: ≥ 20.10
- **支持架构**：x86_64 / ARM64

## 5. 技术栈建议

> 仅供参考，开发团队可根据实际情况调整

### 5.1 前端
- **框架**: React 18 + TypeScript
- **UI 库**: Ant Design 5 / Shadcn UI
- **图表**: ECharts（实时图表、关系图、热力图）
- **状态管理**: Zustand
- **实时通信**: WebSocket（Socket.IO）
- **大屏**: DataV / 自研大屏组件

### 5.2 后端
- **语言**: Python 3.11+（跨平台优势、AI 生态）
- **框架**: FastAPI（高性能异步、自动文档）
- **任务队列**: Celery + Redis（Round 调度、Check、评分）
- **WebSocket**: Socket.IO / FastAPI WebSocket
- **CLI**: Click / Typer

### 5.3 数据库
- **关系型**: PostgreSQL（赛事、用户、积分、日志）
- **缓存**: Redis（排行榜、会话、实时数据）
- **时序数据**: ClickHouse（可选，流量分析）

### 5.4 容器管理
- **Docker SDK for Python**（容器生命周期管理）
- **Docker Compose**（本地开发）
- **网络**: Docker Bridge Network（队伍隔离）

### 5.5 AI/智能
- **本地模型**: Ollama + Qwen2.5（离线 AI 分析）
- **规则引擎**: 自研轻量级规则引擎（WAF / 异常检测）
- **流量分析**: Scapy + 自研解析器

### 5.6 DevOps
- **打包**: Docker（平台本身也容器化）
- **CI/CD**: GitHub Actions
- **配置管理**: YAML 配置文件

## 附录 B：竞赛计分规则示例

```
攻击得分 = 成功提交他人 Flag 数量 × 攻击权重
防御得分 = Check 通过次数 × 防御权重
总积分 = 攻击得分 + 防御得分 - 罚分
罚分 = 被攻破次数 × 被攻破惩罚 + 恢复次数 × 恢复惩罚
```

默认权重（可配置）：
- 攻击权重：10 分/次
- 防御权重（Check 通过）：5 分/轮
- 被攻破惩罚：20 分/次
- 恢复惩罚：5 分/次
